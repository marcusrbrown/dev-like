# Astro component scripts and styles are bundled per-page, not site-wide

## Context

Registry pages were generated by a script that emitted raw `<button data-copy-button ...>`
HTML, assuming `CopyCommand.astro`'s `initCopyButtons()` script would bind to it. Astro
bundles a component's `<script>` only into pages that render that component. Registry pages
never rendered `CopyCommand.astro`, so the button was dead — no copy, no analytics event —
and its borrowed `.copy-button` CSS broke because it lacked the component's wrapper
positioning context. Caught by inspecting compiled `dist/` output, not by reading source.

## Guidance

- Raw HTML emitted outside a component cannot rely on any component's script or scoped CSS —
  Astro's per-page script/style bundling only follows actual component usage in that page's
  render tree.
- Fix (PR #22): a site-wide delegated document-level click handler, injected once via a
  Starlight `Head` component override (render Starlight's default `Head` plus one additional
  script), paired with self-contained `.registry-copy-button` CSS using static positioning
  instead of inheriting the component's wrapper context.
- If a behavior needs to work across many generated pages, make it global (document-level
  delegation, site-wide script) rather than assuming component code will "just be there."

## Why This Matters

- The failure mode is silent: no build error, no console error at generation time — a
  visually correct-looking button that simply does nothing when clicked.
- Borrowing a component's CSS classes without its markup wrapper produces subtly broken
  layout, not an obvious visual break.

## When to Apply

- Any generator or script that emits HTML resembling a component's output without rendering
  the component itself.
- Any interactive markup meant to work across many generated/dynamic pages — verify it isn't
  implicitly depending on a single component instance elsewhere in the site.

## Examples

```astro
---
// src/components/Head.astro — Starlight override
import Default from '@astrojs/starlight/components/Head.astro'
---
<Default><slot /></Default>
<script>
  document.addEventListener('click', (e) => {
    const btn = e.target.closest('[data-copy-button]')
    if (btn) { /* copy + analytics */ }
  })
</script>
```

## Prevention

- Verify interactive behavior by clicking in the rendered site and checking clipboard/network
  readback — not by grepping markup for the right attributes.
- Inspect compiled `dist/` output when debugging "the markup looks right but doesn't work"
  issues; source-level review won't show per-page bundling gaps.
